## Deviations from design

1. **Line count**: 491 vs design estimate of ~300. The 400-line review
   budget from the orchestrator preflight is exceeded by 91 lines.
   The implementation matches the design's structure (header +
   constants + 4 arrays + 8 helpers + main) but the prose comments
   and per-section banners (matching `setup-omarchy`'s style) push
   the total above the budget. **The orchestrator should ask the
   user whether to trim further (e.g., remove section banners +
   shorten header) or accept the overage.** The code is correct
   and lint-clean; this is a review-scope question, not a defect.

2. **`is_installed` accepts both singular and plural** category
   names. The design's idempotency table uses plural, but the
   actual data flow has `build_plan` emit singular (the log key is
   `stub` / `pkg` / `webapp` / `tui` per REQ-CLEANUP-005), and
   those rows are passed straight to `is_installed`. Accepting
   both makes the function tolerant of either form and avoids a
   mismatch in any future caller.

3. **`is_installed packages` tolerates a missing `pacman`**
   (returns 1, treated as "not installed"). This makes `--list`
   and `--dry-run` portable to non-Arch test environments.
   Real-run callers in `execute_plan` require `pacman` explicitly
   via `require_command` before the loop. Matches the design's
   "Error Handling Strategy" section.
