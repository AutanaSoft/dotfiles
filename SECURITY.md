# Security Policy

## Scope

This repository contains personal dotfiles, shell helpers, and configuration templates for Omarchy Linux and WSL2 Fedora. The exposed attack surface is limited to shell scripts and dotfile content; there is no hosted service or user-facing input to exploit.

## Supported Versions

Only the `main` branch is maintained. Older branches, forks, and tags receive no security fixes.

## Reporting a Vulnerability

Report security issues privately through one of these channels:

- **GitHub Security Advisories** (preferred): open a [private advisory](../../security/advisories/new). This keeps the report off the public timeline until disclosure.
- **Email**: open a GitHub issue requesting a private contact channel if you cannot use advisories.

Do not file a public issue for suspected vulnerabilities. Public reports make coordinated disclosure harder and may put users at risk before a fix is available.

## Response

This is a personal project maintained in spare time. Expect a best-effort response within a reasonable window, but there is no guaranteed SLA.
